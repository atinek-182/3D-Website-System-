# Bad Website References And Patterns

## Purpose

This file stores websites and design patterns I dislike so agents avoid repeating them.

## How To Use This File

* Add weak websites or weak patterns.
* Be specific about what is bad.
* Do not just write "bad design".
* Explain the better alternative.

## Global Reference Rules

1. References are for extracting principles, not copying designs.
2. Do not copy layouts directly.
3. Do not copy brand identity.
4. Do not copy text, assets, or visuals.
5. Extract:
   * layout principle
   * typography principle
   * motion principle
   * interaction principle
   * 3D/canvas principle
   * color/lighting principle
   * performance lesson
6. Every reference must include:
   * what I like
   * what I dislike
   * what to adapt
   * what to avoid
7. If a reference is not analyzed, do not use it as a design rule.
8. If a reference is old or changed, mark it for re-review.

## Common Bad 3D Website Patterns

* random floating objects
* particles with no meaning
* unreadable text over canvas
* fake glassmorphism everywhere
* excessive glow/bloom
* heavy model with no purpose
* scroll hijacking
* no mobile fallback
* long loading with weak payoff
* generic black-gradient SaaS hero

## Common Bad Motion Patterns

* animations delaying content
* random camera shake
* overused parallax
* too many things moving at once
* no reduced-motion consideration
* scroll snapping that feels forced
* hover effects that hide usability
* page transitions longer than needed

## Common Bad Typography Patterns

* tiny low-contrast text
* too many fonts
* weak heading hierarchy
* generic Inter-only layout
* huge text with no spacing control
* all-caps everywhere
* decorative type that hurts readability
